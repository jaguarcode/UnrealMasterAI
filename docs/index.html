How It Works

From Natural Language to Engine Changes

Describe your intent. Claude orchestrates the tools. Unreal executes in real time.

1

Describe Intent

"Add a PrintString node to BP_TestActor after BeginPlay, set the message to Hello World."

2

Serialize & Plan

Claude calls blueprint-serialize to understand the current graph, then plans the node creation sequence.

3

Execute Changes

Creates the PrintString node, connects exec pins, sets the default value — all via C++ engine APIs on the GameThread.

4

Compile & Heal

Triggers Live Coding. If compilation fails, parses errors, applies fix, and retries up to 3 times automatically.

# ue-plugin/Content/Python/uma/actor_spawn.py import unreal from uma.utils import execute_wrapper, make_result, get_required_param @execute_wrapper def execute(params): class_path = get_required_param(params, "className", str) location = params.get("location", {"x": 0, "y": 0, "z": 0}) actor_class = unreal.load_class(None, class_path) loc = unreal.Vector(location["x"], location["y"], location["z"]) actor = unreal.EditorLevelLibrary.spawn_actor_from_class(actor_class, loc) return make_result( name=actor.get_actor_label(), className=actor.get_class().get_name() )

Trust & Safety

3-Tier Safety System

Every tool is classified by risk level. Destructive operations require explicit human approval.

Safe

31 tools — Read-only operations. Query data, list assets, inspect Blueprints. No approval needed.

Warn

26 tools — Recoverable mutations. Create assets, spawn actors, modify properties. Logged but not gated.

Dangerous

7 tools — Destructive operations. Delete assets, cook content, exec console commands. Requires Slate dialog approval with 60s timeout.
